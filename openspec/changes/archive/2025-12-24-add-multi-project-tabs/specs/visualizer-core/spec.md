## 新增需求

### 需求：STORY-000001 (多项目页签)
系统必须支持同时打开多个项目页签，并在页签间切换。

#### 场景：新建空白页签
- **当** 用户触发“新建页签”
- **那么** 系统创建一个空白页签
- **并且** 该页签的项目状态、搜索状态、终端会话均为全新上下文

#### 场景：页签数量上限
- **当** 用户尝试新建第 6 个页签
- **那么** 系统提示“最多只能打开 5 个页签”
- **并且** 不创建新页签

#### 场景：关闭页签释放资源
- **当** 用户关闭某个页签
- **那么** 系统释放该页签对应的目录监听与终端会话资源

### 需求：STORY-000002 (菜单-打开最近)
系统必须在菜单提供“打开最近”，并按规则在页签中打开。

#### 场景：当前页签已有项目则新建页签打开
- **当** 当前页签已加载项目且用户选择“打开最近”的某个项目
- **那么** 系统新建页签并在新页签中打开该项目

#### 场景：当前页签为空白则在当前页签打开
- **当** 当前页签未加载项目且用户选择“打开最近”的某个项目
- **那么** 系统在当前页签中打开该项目

### 需求：STORY-000003 (菜单-新建页签)
系统必须在菜单提供“新建页签”，用于创建空白页签。

#### 场景：通过菜单新建页签
- **当** 用户从菜单触发“新建页签”
- **那么** 系统创建一个空白页签并切换到该页签
