## 新增需求
### 需求：目录树按需加载
系统必须支持目录树按需加载或限制递归深度，以降低首次加载成本。

#### 场景：首次加载大型项目
- **当** 项目满足大型项目口径
- **那么** 初次加载只需加载必要层级
- **并且** 后台应自动完成全量加载
- **并且** 首次加载时间满足性能目标

#### 场景：用户展开节点
- **当** 用户展开未加载的目录节点
- **那么** 系统应加载该节点的直接子级

#### 场景：后台全量加载完成
- **当** 后台完成全量加载
- **那么** 树视图应完整可浏览，不需要用户手动触发

#### 场景：后台加载提示
- **当** 后台正在加载全量树
- **那么** 界面应显示非侵入式提示（如角落状态提示）
- **并且** 提示样式与当前页面风格一致，小字、无边框、现代感

### 需求：文件变更触发合并
系统必须合并短时间内的多次文件变更，避免重复全量重载。

#### 场景：短时间多次变更
- **当** 1 秒内发生多次文件变化
- **那么** 项目重载触发次数不应超过 1 次

### 需求：任务进度按需与限速
系统必须在任务进度扫描时限制并发，并支持按需触发。

#### 场景：任务进度扫描
- **当** 需要展示变更进度
- **那么** tasks.md 读取并发应受限

### 需求：搜索过滤防抖与缓存
系统必须对搜索过滤做防抖，并复用上一次结果缓存。

#### 场景：连续输入
- **当** 用户连续输入搜索关键词
- **那么** 系统应在短暂稳定后再执行过滤
- **并且** 对重复查询应复用缓存结果

#### 场景：搜索覆盖全量
- **当** 用户执行搜索
- **那么** 搜索结果必须覆盖项目全量文件

### 需求：树渲染虚拟化或分页
系统必须在节点数量较大时避免一次性渲染全部节点。

#### 场景：节点数量过多
- **当** 节点数超过阈值
- **那么** 系统应采用虚拟化或分页方式渲染

### 需求：Markdown 与高亮按需
系统必须对 Markdown 渲染与代码高亮采取按需策略。

#### 场景：打开大文件
- **当** 打开大型 Markdown 或代码文件
- **那么** 高亮与渲染应延迟或分段执行
