## 新增需求
### 需求：项目选择与最近项目
系统必须允许用户选择项目目录，并展示最近打开的项目列表。

#### 场景：选择项目目录
- **当** 用户点击选择项目并确认目录
- **那么** 应加载该目录并记录到最近列表

### 需求：文件树浏览
系统必须展示 changes/ 与 specs/ 下的文件树，并支持展开折叠。

#### 场景：浏览文件树
- **当** 用户展开文件夹并点击文件
- **那么** 应加载并显示该文件内容

### 需求：内容渲染
系统必须渲染 Markdown，并对代码块进行语法高亮。

#### 场景：打开 Markdown 文件
- **当** 用户打开 .md 文件
- **那么** 应显示格式化内容与高亮代码块

### 需求：搜索与过滤
系统必须提供搜索输入框用于过滤左侧文件树。

#### 场景：输入搜索关键字
- **当** 用户输入关键字
- **那么** 文件树只显示匹配项

### 需求：主题与偏好
系统必须支持亮色、暗色与跟随系统主题，并持久化偏好。

#### 场景：切换主题
- **当** 用户切换主题
- **那么** 主题立即生效并保存设置

### 需求：终端入口
系统必须提供右侧终端开关与连接状态提示。

#### 场景：打开终端面板
- **当** 用户点击终端按钮
- **那么** 终端面板显示并可操作

### 需求：OpenSpec 分组展示
系统必须将 changes/ 与 specs/ 作为独立分组展示，其余文件作为根级条目展示。

#### 场景：加载包含 changes 与 specs 的项目
- **当** 项目目录包含 changes/ 与 specs/
- **那么** 文件树显示 Changes 与 Specs 分组，并保留其他文件

### 需求：归档标记
系统必须标记包含 archive 的变更条目，以提示已归档状态。

#### 场景：浏览归档变更
- **当** 用户展开包含 archive 的变更目录
- **那么** 该条目显示为归档状态

### 需求：任务进度徽章
系统必须解析 tasks.md 并显示完成进度。

#### 场景：打开包含任务清单的变更
- **当** tasks.md 内包含勾选与未勾选项
- **那么** 变更条目显示完成数与总数

### 需求：文件变更刷新
系统必须监听项目目录的变更并刷新文件树与内容。

#### 场景：文件被编辑或新增
- **当** 项目目录发生文件变更
- **那么** 文件树自动刷新并保持可用

### 需求：图片文件显示
系统必须支持图片文件的预览显示。

#### 场景：打开图片文件
- **当** 用户打开 .png/.jpg/.svg 等图片
- **那么** 内容区显示对应图片

### 需求：错误提示
系统必须在读取失败或非桌面环境时提示错误原因。

#### 场景：非桌面环境访问项目
- **当** 用户在浏览器环境尝试选择项目
- **那么** 系统提示该功能仅桌面可用
