## 修改需求
### 需求：一键收缩与主视图切换
系统必须在顶部工具栏提供文件面板与任务面板的显示开关按钮。

#### 场景：隐藏文件面板
- **当** 用户点击文件面板按钮关闭
- **那么** 文件面板隐藏并保留上次宽度
- **并且** 不影响任务面板与终端面板状态

#### 场景：隐藏任务面板
- **当** 用户点击任务面板按钮关闭
- **那么** 任务面板隐藏并保留上次宽度
- **并且** 不影响文件面板与终端面板状态

#### 场景：重新打开面板
- **当** 用户再次点击对应按钮开启
- **那么** 面板恢复显示并使用上次宽度

### 需求：文件面板与任务面板联动
系统必须允许文件面板与任务面板独立显示，不强制联动。

#### 场景：仅隐藏文件面板
- **当** 文件面板关闭且任务面板开启
- **那么** 任务面板保持显示

#### 场景：仅隐藏任务面板
- **当** 任务面板关闭且文件面板开启
- **那么** 文件面板保持显示

## 新增需求
### 需求：终端按钮仅隐藏显示
系统必须允许通过顶部终端按钮隐藏/显示终端面板，而不重启终端会话。

#### 场景：隐藏终端面板
- **当** 用户点击顶部终端按钮关闭
- **那么** 终端面板隐藏但会话保持运行

#### 场景：再次显示终端面板
- **当** 用户再次点击顶部终端按钮
- **那么** 终端面板恢复显示并保持当前会话

### 需求：分割条仅用于调整宽度
系统必须仅通过分割条拖拽调整面板宽度，分割条不提供收起/展开按钮或点击事件。

#### 场景：拖拽分割条
- **当** 用户拖拽分割条
- **那么** 相邻面板宽度随拖拽变化并保持记录

#### 场景：点击分割条
- **当** 用户点击分割条
- **那么** 不触发面板收起或展开

### 需求：面板宽度保护
系统必须避免面板宽度被拖拽到 0px。

#### 场景：拖拽到最小宽度
- **当** 用户持续向内拖拽分割条
- **那么** 面板宽度停在最小保护值且可反向拖回
